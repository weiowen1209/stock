# 页面操作流程自动化测试方案

## 1. 测试目标

本方案用于验证人形机器人产业 A 股分析仪表板的核心页面流程，覆盖用户从打开首页、查看产业链、选择股票、同步当前股票、查看基本面和技术面、进入个股档案、导入数据并确认入库的主路径。

测试重点不是验证外部行情源的真实市场数据，而是验证页面操作、前端状态、后端 API、SQLite 入库和刷新机制是否形成闭环。

## 2. 测试范围

- 首页顶部与数据状态面板
- 产业链总览页签
- 基本面深度分析页签
- 技术面分析页签
- 个股深度档案页签
- 导入工作台页签
- 同步当前股票链路
- 错误与空数据兜底

## 3. 自动化测试策略

当前项目已有后端 pytest 和前端构建检查，前端暂未引入 Playwright/Vitest。因此本轮自动化测试采用两层方式：

1. 接口与数据机制测试：通过 FastAPI ASGI 客户端直接调用 API，验证页面依赖的数据结构、同步结果、导入结果和状态刷新。
2. 前端静态契约测试：读取 Vue 页面源码，验证页面必须具备的页签、按钮、表格列、图表标题、提示文案和 API 触发入口。

后续如果引入浏览器端测试框架，可以在本方案基础上升级为真正的端到端点击测试。

## 4. 测试数据约定

- 默认测试股票：688017 绿的谐波
- 默认产业链：核心零部件
- 默认报告期：2030年报
- 默认报告日期：2030-12-31
- 同步测试使用测试 Provider，避免依赖真实外部行情源波动。

## 5. 测试用例

### TC-01 首页初始化与全局数据状态

用户操作：

1. 打开首页。
2. 等待页面初始化完成。
3. 查看顶部标题、数据来源、数据状态面板和默认页签。

机制说明：

1. 前端挂载后执行 `loadInitialData()`。
2. 并发请求 `/api/stocks`、`/api/stocks/by-industry`、`/api/quotes`、`/api/sync/status`。
3. 如果当前未选择股票，则默认选择股票池第一只股票。
4. 随后加载当前股票 K 线、基本面、估值和技术指标。

预期结果：

- 页面标题为“机器人产业分析仪表板”。
- 顶部不再重复同步按钮，同步入口统一放在数据状态面板中。
- 数据状态面板展示“数据链路正常”或“缺失 N 项”。
- 默认页签为“产业链总览”。
- 股票池不少于 20 只。
- 产业链分组不少于 6 个。
- `/api/sync/status` 包含缺失行情、缺失 K 线、Provider 健康状态和同步进度。
- 如果出现缺失数据，应该切换数据源进行同步，直到补齐缺失数据。

测试自动化断言：

- API 返回统一结构：`data`、`meta`、`error`。
- 股票池数量不少于 20。
- 产业链分组数量不少于 6。
- 状态接口包含 `missing_quotes`、`missing_kline`、`providers`、`progress`。
- `App.vue` 包含标题、同步按钮、数据状态面板和 5 个页签。

### TC-02 同步当前股票

用户操作：

1. 在产业链总览中选择 `688017 绿的谐波`。
2. 点击数据状态面板中的“同步当前股票”。
3. 等待同步完成。

机制说明：

1. 前端调用 `store.syncCurrentStock()`。
2. 当前股票代码为空时不触发同步。
3. 当前股票代码存在时，POST `/api/sync/trigger`，请求体包含 `codes`、`include_quotes`、`include_kline` 和 `period`。
4. 后端先执行 `sync_quotes()`，写入或更新 `realtime_quotes`，写入 `sync_log`，并推送 quotes 阶段进度。
5. 后端再执行 `sync_kline()`，写入或更新 `kline_data`，写入 `sync_log`，并推送 done 阶段进度。
6. 前端同步后刷新行情、同步状态、K 线、基本面和技术指标。

预期结果：

- 同步请求返回 realtime 和 kline 两类结果。
- 两类结果状态为 `success`，或在部分 Provider 失败时为 `fallback`。
- 行情表存在当前股票记录。
- K 线表存在当前股票日线记录。
- 最新同步日志包含 realtime 和 kline。
- 同步进度最终进入 done 阶段或 percent 达到 100。

自动化断言：

- `/api/sync/trigger` 返回两个任务结果。
- `task_type` 包含 `realtime` 和 `kline`。
- `/api/quotes?codes=688017` 返回当前股票行情。
- `/api/kline/688017` 返回当前股票 K 线。
- `/api/sync/status` 返回同步日志和进度。
- `stockStore.ts` 中同步后刷新 quotes、sync status 和当前股票分析数据。

### TC-03 产业链总览

用户操作：

1. 打开“产业链总览”。
2. 查看产业链按钮、指标卡和股票表格。
3. 点击“核心零部件”。
4. 点击 `688017 绿的谐波` 股票行。
5. 点击“全部”恢复全量列表。

机制说明：

1. 产业链按钮来自 `/api/stocks/by-industry`。
2. 指标卡分别显示股票池规模、产业链环节、行情记录。
3. 股票表格数据来自 `filteredStocks`。
4. 点击股票行触发 `selectStock(code)`，刷新当前股票分析数据。

预期结果：

- 页面显示“产业链雷达”和“人形机器人A股股票池”。
- 指标卡包含“股票池规模”“产业链环节”“行情记录”。
- 表格列包含代码、名称、产业链、细分定位、涨跌幅、最新价。
- 点击“核心零部件”后，列表只展示该产业链股票。
- 该分组中应包含 `688017 绿的谐波`。
- 点击股票后，当前股票切换为 `688017`。

自动化断言：

- `/api/stocks/by-industry` 中每个分组包含 `industry_chain` 和 `stocks`。
- 分组内股票的 `industry_chain` 与分组名一致。
- 核心零部件分组包含 `688017`。
- `IndustryOverview.vue` 包含指标卡和股票表格。
- `StockTable.vue` 表格列满足页面预期，并通过 row-click 触发 select。

### TC-04 基本面深度分析

用户操作：

1. 选择 `688017 绿的谐波`。
2. 切换到“基本面深度分析”。
3. 查看指标卡和图表。

机制说明：

1. 页面依赖 `/api/fundamentals/688017`、`/api/fundamentals/688017/segments`、`/api/fundamentals/688017/expenses` 和 `/api/valuation/688017`。
2. 财报数组最后一条作为 latest 指标来源。
3. 图表分别展示营收与净利润趋势、业务分部收入、费用结构和估值指标。

预期结果：

- 指标卡包含营业收入、净利润、毛利率、研发费用率。
- 图表标题包含营收与净利润趋势、业务分部收入、费用结构、估值指标。
- 默认种子数据下，`688017` 应存在财报和估值数据。
- 空数据情况下页面应展示 `--` 或空图表，而不是崩溃。

自动化断言：

- 基本面相关 API 返回统一结构。
- `688017` 财报数据不少于 1 条。
- `688017` 估值数据不少于 1 条。
- `Fundamental.vue` 包含所有指标卡和图表标题。

### TC-05 技术面分析

用户操作：

1. 选择 `688017 绿的谐波`。
2. 点击“同步当前股票”，确保 K 线数据存在。
3. 切换到“技术面分析”。
4. 查看 K 线、均线、MACD 和 RSI6。

机制说明：

1. K 线图使用 `/api/kline/688017?period=day`。
2. 技术指标使用 `/api/technical/688017?period=day`。
3. 后端根据 K 线计算 MA5、MA10、MA20、MACD、signal、histogram 和 RSI6。
4. K 线为空时，技术指标日期为空，接口 meta.stale 为 true。

预期结果：

- 页面显示技术走势 K 线图。
- 页面显示“均线系统”“MACD”“RSI6”。
- 技术指标数组长度与日期数组一致。
- 空数据时页面不崩溃。

自动化断言：

- `/api/technical/688017` 返回 dates、close、ma5、ma10、ma20、macd、signal、histogram、rsi6。
- 非空技术指标下，各主要数组长度与 dates 一致。
- `Technical.vue` 包含 K 线、均线、MACD 和 RSI6 标题。

### TC-06 个股深度档案

用户操作：

1. 在产业链总览中点击 `688017 绿的谐波`。
2. 切换到“个股深度档案”。
3. 查看档案卡、行情指标和 K 线图。

机制说明：

1. 档案信息来自当前股票 `currentStock`。
2. 行情指标来自 `currentQuote`。
3. K 线图来自 `store.kline`。
4. `supply_chain_tags` 按 JSON 解析为标签。

预期结果：

- 档案卡显示“绿的谐波”和 `688017`。
- 产业链标签显示“核心零部件”。
- 细分定位显示“谐波减速器”。
- 指标卡包含最新价、换手率、成交额、总市值。
- K 线图标题包含股票名称和代码。
- 标签 JSON 解析失败时页面不崩溃。

自动化断言：

- `/api/stocks` 中存在 `688017` 且产业链为核心零部件。
- 股票细分定位包含谐波减速器。
- `StockDetail.vue` 包含档案字段、行情指标和 K 线图。

### TC-07 导入工作台：上传文件预览

用户操作：

1. 切换到“导入工作台”。
2. 不选择文件，点击“生成预览”。
3. 选择 `.txt` 测试文件。
4. 点击“生成预览”。

机制说明：

1. 未选择文件时前端提示“请先选择文件”，不调用后端。
2. 选择文件后调用 `/api/imports/upload`。
3. 后端创建导入批次并返回预览。
4. 前端展示批次号、可信度、财务字段、业务分部和 warnings。
5. 预览成功后刷新 `/api/imports`。

预期结果：

- 未选择文件时提示“请先选择文件”。
- 选择文件后提示“PDF文件已选择”。
- 生成成功后提示“预览已生成”。
- 预览区显示批次号、股票代码、报告期、营业收入、净利润、毛利率、ROE。
- 批次列表新增 upload 类型记录。

自动化断言：

- `ImportWorkbench.vue` 包含未选文件保护逻辑。
- `/api/imports/upload` 返回 batch、financial、segments、expenses、warnings、confidence。
- `/api/imports` 包含刚创建的批次。

### TC-08 导入工作台：手工录入预览与确认入库

用户操作：

1. 切换到“导入工作台”。
2. 填写股票代码、报告期、报告日期、营业收入、净利润、毛利率、ROE、研发费用率。
3. 点击“生成手工预览”。
4. 点击“确认入库”。
5. 回到“基本面深度分析”查看数据。

机制说明：

1. 生成手工预览调用 `/api/imports/manual`。
2. 确认入库调用 `/api/imports/{batch_id}/confirm`。
3. 后端将财务数据写入业务表，并更新批次状态。
4. 前端刷新导入批次列表。

预期结果：

- 生成预览后显示批次号和填写的财务字段。
- 确认后提示“导入已确认入库”。
- 批次状态变为已确认。
- `/api/fundamentals/688017` 能查到新增报告期。

自动化断言：

- `/api/imports/manual` 返回 `meta.source = manual`。
- `/api/imports/{batch_id}/confirm` 返回确认结果。
- 确认后基本面接口能查询到对应报告期数据。
- `ImportWorkbench.vue` 包含生成手工预览、确认入库和刷新批次逻辑。

### TC-09 导入工作台：批次刷新

用户操作：

1. 切换到“导入工作台”。
2. 点击“刷新”。

机制说明：

1. 前端调用 `/api/imports`。
2. 批次表展示 ID、类型、代码、报告期、状态和创建时间。

预期结果：

- 批次表列完整。
- 有导入记录时按后端返回展示。
- 无导入记录时页面不报错。

自动化断言：

- `/api/imports` 返回统一结构。
- `ImportWorkbench.vue` 包含刷新按钮和批次表列。

### TC-10 错误与空数据兜底

用户操作：

1. 模拟某个页面依赖 API 返回空数据或错误。
2. 切换到相关页签。

机制说明：

1. store 中加载函数捕获异常后设置 `store.error`。
2. 相关数据置为空数组或 null。
3. 图表组件应接受空数组。
4. 导入失败使用错误提示。

预期结果：

- API 错误时页面出现明确错误提示。
- 基本面或技术面空数据时展示 `--` 或空图表。
- 页面不白屏，不出现未捕获异常。

自动化断言：

- store 源码包含错误捕获和兜底赋值。
- 页面源码包含导入失败提示。
- 空数据接口仍返回统一结构和 stale 标识。

powershell -ExecutionPolicy Bypass -File scripts/run_checks.ps1

- 页面流程测试全部通过。
