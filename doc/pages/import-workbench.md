# 导入工作台页面开发文档

## 对应代码

- 页面文件：`frontend/src/views/ImportWorkbench.vue`
- API入口：`frontend/src/api/index.ts`
- 类型定义：`frontend/src/api/types.ts`
- 后端接口：`/api/imports/upload`、`/api/imports/manual`、`/api/imports/{batch_id}/confirm`、`/api/imports`

## 页面目标

导入工作台用于完成财务数据的半自动导入闭环。用户可以上传PDF、TXT、CSV等文件生成解析预览，也可以手工录入核心财务指标，确认无误后写入后端财务、业务分部和费用表。

## 布局结构

1. PDF/文本上传卡片：选择文件、填写可选股票代码和报告期、生成预览。
2. 手工录入卡片：填写股票代码、报告期、报告日期、营业收入、净利润、毛利率、ROE、研发费用率等核心字段。
3. 人工确认卡片：展示批次ID、可信度、警告、财务字段和业务分部，并提供确认入库按钮。
4. 导入批次列表：展示历史导入批次ID、类型、股票代码、报告期、状态和创建时间。

## 数据依赖

- `selectedFile`：当前选择的上传文件。
- `uploadCode`：上传文件辅助股票代码。
- `uploadPeriod`：上传文件辅助报告期。
- `manual`：手工录入请求体。
- `preview`：当前导入预览结果。
- `batches`：导入批次列表。
- `loading`：上传、预览、确认过程的统一loading状态。

## 用户交互

- 用户选择文件后，应显示“PDF文件已选择”。
- 点击“生成预览”时，如果没有文件，应提示“请先选择文件”。
- 上传预览成功后，应显示“预览已生成”并刷新导入批次列表。
- 上传预览失败时，应显示后端错误或“生成预览失败”。
- 手工预览成功后，应显示“预览已生成”并刷新导入批次列表。
- 确认入库成功后，应显示“导入已确认入库”并刷新导入批次列表。
- 批次加载失败时，应显示“导入批次加载失败”。

## 错误和空状态

- 没有预览结果时，人工确认区域显示空态。
- 文件解析出的字段可能为空，页面显示 `--`，不阻塞预览展示。
- 扫描版PDF可能无法识别关键指标，当前MVP仍允许生成低可信度预览。
- 所有异步请求必须有 `catch` 提示，不能静默失败。

## 修改规则

- 修改上传字段前，必须先更新本文件“布局结构”和“数据依赖”。
- 修改提示文案前，必须先更新“用户交互”和“错误和空状态”。
- 修改确认入库payload前，必须同步更新 `ConfirmImportRequest` 类型和后端导入Schema。
- 新增导入字段前，必须同步更新后端 `import_service.py`、前端类型、本文件和相关测试。

## 验证要求

- `npm run build` 必须通过。
- `python -m pytest tests/test_stage7_delivery.py::test_import_workbench_surfaces_upload_failures` 必须通过。
- 选择文件后必须出现选择成功提示。
- 后端不可用时，点击生成预览必须显示错误提示，不能无反应。
