# 个股深度档案页面开发文档

## 对应代码

- 页面文件：`frontend/src/views/StockDetail.vue`
- 主要组件：`MetricCard.vue`、`KLineChart.vue`
- 状态来源：`frontend/src/stores/stockStore.ts`
- 后端接口：`/api/quotes`、`/api/kline/{code}`

## 页面目标

个股深度档案页面用于集中展示当前股票的基本信息、实时行情指标和K线走势，是单只股票研究的概览页。

## 布局结构

1. 个股基础信息区域：展示名称、代码、产业链、细分环节等信息。
2. 行情指标卡区域：展示当前价格、涨跌幅、成交额等行情信息。
3. K线图区域：展示当前股票K线走势。

## 数据依赖

- `store.currentStock`：股票基础信息。
- `store.currentQuote`：当前股票实时行情。
- `store.kline`：当前股票K线数据。
- `store.currentCode`：当前股票代码。

## 用户交互

- 页面跟随全局当前股票，不提供独立选股控件。
- 用户通过产业链总览选择股票后，本页自动展示对应个股档案。
- 用户点击同步当前股票后，行情和K线应刷新。

## 错误和空状态

- 没有当前股票时，应显示空态或兜底内容，不应访问空对象字段导致崩溃。
- 行情缺失时，指标卡显示 `--` 或其他兜底值。
- K线为空时，图表区域应保持稳定展示。

## 修改规则

- 新增行情指标前，必须先更新本文件“布局结构”和“数据依赖”。
- 修改个股字段展示前，必须同步检查后端 `Stock` Schema 和前端 `Stock` 类型。
- 修改K线图展示前，必须同步更新 `doc/pages/technical.md` 中相关K线说明。

## 验证要求

- `npm run build` 必须通过。
- 当前股票切换后，基础信息、行情指标和K线应同步变化。
- 当前股票或行情为空时页面不应崩溃。
